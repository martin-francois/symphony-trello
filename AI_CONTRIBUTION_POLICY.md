# AI Contribution Policy

AI-assisted contributions are welcome when they are transparent, reviewable, and owned by a human.

## Expectations

- A human contributor is responsible for the design, code, tests, docs, and licensing of every
  submitted change.
- Disclose material AI assistance in the pull request body. Short completion, search, formatting, or
  typo fixes do not need a detailed disclosure.
- Review generated code before committing it. Do not submit code you cannot explain or maintain.
- Run the relevant validation locally and report the commands in the pull request.
- Do not paste secrets, Trello credentials, Codex auth files, private board links, private host
  paths, or proprietary third-party code into AI tools.
- Do not add generated dependencies, vendored code, or copied snippets unless their license and
  provenance are clear and compatible with this repository.
- Keep AI-generated pull requests focused. Separate broad refactors from behavior changes when
  practical.

## Maintainer Review

Maintainers review AI-assisted changes by the same standard as any other contribution. Disclosure is
not a substitute for tests, documentation, or careful review.
